LZ4 概述

什么是 LZ4？

LZ4 是由 Yann Collet 设计的无损压缩算法，专注于压缩和解压的极致速度。它属于 LZ77 算法家族。

核心特性

  • 压缩速度：>500 MB/s 每核（默认级别）
  • 解压速度：数 GB/s 每核（接近内存带宽极限）
  • 压缩比：约 2.1:1（取决于数据类型）
  • 内存占用：压缩 16KB（可配置），解压极低
  • 线程安全：是，支持多线程压缩 (v1.10.0+)

工作原理

LZ4 使用基于哈希的方法在已处理数据中查找匹配序列：

  1. 维护 64 KB 的滑动窗口用于匹配搜索
  2. 4 字节哈希表实现 O(1) 匹配查找
  3. 每个序列由以下组成：token + literal_length + literals + offset + match_length
  4. 不使用熵编码 — 解压纯靠内存拷贝

数据块

每个块有 4 字节大小头（小端序）。如果最高位（0x80000000）被设置，则块以未压缩方式存储。其余 31 位为数据大小。

头部校验和

一个字节：(xxh32(描述符) >> 8) & 0xFF。覆盖从 FLG 到可选字段的所有描述符字节。

结束标记

0x00000000 — 标记块序列结束。

LZ4 块格式规范

块结构

LZ4 压缩块由多个序列组成。每个序列包含一个 token、可选的字面量长度字节、字面量、2 字节偏移量和可选的匹配长度字节。

Sequence = Token + [Extra Literal Length] + Literals + Offset(2B) + [Extra Match Length]

Token 字节

一个字节分为两个 4 位字段：

  • 高 4 位：字面量长度 (0-15)。如果为 15，读取额外字节（每个 0-255，累加直到 <255）
  • 低 4 位：匹配长度减去 MIN_MATCH(4)。值 0 表示 4 字节，15 表示 19+ 字节（需读取额外字节）

偏移量

2 字节，小端序。表示从多远的位置开始拷贝。值 0 无效。最大值：65535。

编码示例

场景Token额外字节结果
3 个字面量，匹配=50x31无3 字节字面量，从偏移位置拷贝 5 字节
20 个字面量，匹配=40xF00x0515+5=20 字节字面量，拷贝 4
0 个字面量，匹配=1000x5F0x510 字面量，4+15+81=100 字节拷贝

块结束规则

  • 最后一个序列仅包含字面量（无偏移量）
  • 输入的最后 5 字节始终为字面量
  • 最后一个匹配必须在块结束前至少 12 字节开始
  • < 12 字节的块无法压缩

重叠拷贝（类 RLE）

当 matchLength > offset 时，匹配延伸到当前写入位置之后。通过逐字节拷贝处理，有效地重复字节。偏移量为 1、matchLength=100 时，最后一个字节重复 100 次。

压缩级别指南

快速模式 (级别 1-2)

使用标准 LZ4 算法和哈希表。加速因子控制速度/压缩比权衡 — 值越大跳过更多位置，速度更快但压缩比更低。

HC 模式 (级别 3-12)

高压缩模式使用更多 CPU 来查找更好的匹配。HC 模式使用基于链的搜索而非简单哈希表。

级别算法速度压缩比说明
1 (默认)Fast最快良好实时场景最佳选择

--fastFast极快较低更高加速因子

关键洞察：解压速度与压缩级别完全相同。解压器不知道也不关心使用了哪个级别。

各级别使用场景

  • 级别 1：默认 — 良好平衡。用于日志文件、网络传输、实时数据
  • 级别 3-9：压缩一次但解压多次（如游戏资源、软件分发）
  • 级别 10-12：归档存储，压缩时间不重要
  • --fast：极端速度场景、嵌入式系统、高吞吐管道
字典压缩

概述

字典压缩对小数据（KB 级别）非常有效，如 JSON 记录、日志行或短消息。预训练字典为压缩器提供"已知前缀"上下文。

工作原理

  1. 从数据的代表性样本构建字典
  2. 在处理前将字典加载到压缩流中
  3. 压缩器使用字典内容作为虚拟历史，实现更好的匹配
  4. 解压器必须使用完全相同的字典

字典 ID

帧格式支持 4 字节字典 ID 字段，帮助解码器选择正确的字典。这是可选的 — 字典也可以通过上下文确定。

构建字典

推荐：使用 Zstandard 字典构建器 — 它也适用于 LZ4。字典应包含数据样本中的常见模式。

CLI 快速参考

基本用法

lz4 file.txt              # Compress → file.txt.lz4
lz4 -d file.txt.lz4       # Decompress → file.txt
lz4 -12 file.txt           # Max compression
lz4 -T4 file.txt           # 4-thread compression
lz4 -t file.lz4            # Test integrity
lz4 --list file.lz4        # Show file info
lz4 -b1 -e12 file.txt      # Benchmark levels 1-12

使用场景与最佳实践

推荐使用场景

  • 日志压缩：快速压缩，即时解压用于日志分析
  • 数据库存储：RocksDB、Cassandra、HBase 使用 LZ4 进行块压缩
  • 网络传输：以最小延迟开销减少带宽
  • 游戏资源：加载画面期间快速解压
  • 内存压缩：Linux zswap/zram 使用 LZ4 进行页面压缩
  • 备份归档：结合 tar 使用：tar cf - dir/ | lz4 > archive.tar.lz4

最佳实践

  1. 除非特别需要更高压缩比，否则使用默认级别 (-1)
  2. 启用 --content-size 以获得更好的工具支持
  3. 对于小数据（<1KB），考虑使用字典压缩
  4. 级别 10+ 且关注解压速度时，使用 --favor-decSpeed
  5. 多线程（-T）仅对大文件有效；小文件不值得使用
